# Проект Django REST API (Домашнее задание)

    Education — это веб-приложение на Django REST Framework, предназначенное для создания, управления и продажи 
    онлайн-курсов и уроков. Платформа поддерживает систему подписок, интеграцию с платежной системой Stripe, а также 
    гибкую систему прав доступа для пользователей, модераторов и администраторов.

## Описание
    Цель проекта
    Создать масштабируемую образовательную платформу, где авторы могут публиковать курсы и уроки, а пользователи — 
    приобретать доступ к ним или подписываться на обновления.
    Цель проекта — продемонстрировать базовые навыки создания REST API с использованием Django и Django REST Framework.

**Основные функции:**
    Управление курсами и уроками: создание, редактирование, удаление.
    Платежная система: оплата курсов и уроков через Stripe.
    Подписки: пользователи могут подписываться на курсы и получать уведомления об обновлениях.
    Гибкие права доступа:
        - Пользователи: могут создавать свои курсы/уроки, оплачивать контент, управлять подписками.
        - Модераторы: могут просматривать и редактировать все курсы/уроки, но не удалять их.
        - Администраторы: полный доступ ко всем функциям.
    REST API: полный набор эндпоинтов для интеграции с фронтендом или мобильными приложениями.
J   WT-аутентификация: безопасный вход и управление сессиями.

## Использование
1. **Запустите сервер разработки:**
   ```bash
   python manage.py runserver
   
2. **API доступно по адресу:**
    http://127.0.0.1:8000/api/redoc
    http://127.0.0.1:8000/swagger/#/
 
3. **Основные возможности API:**
    ```bash
   
    Аутентификация и профиль:
       - Регистрация нового пользователя по email.
       - Вход в систему с получением JWT-токена.
       - Просмотр и редактирование своего профиля.
       - Удаление аккаунта.
   
    Аутентификация и профиль:
       - Создание, редактирование и удаление курсов и уроков.
       - Каждый курс может содержать множество уроков.
       - Уроки включают описание, превью и ссылку на видео (только YouTube).
       - При просмотре курса вы видите:
             количество уроков,
             полный список уроков,
             подписаны ли вы на обновления этого курса.
   
    Подписки:
       - Вы можете подписаться на любой курс, чтобы получать уведомления о новых уроках. 
       - Подписка работает как переключатель: повторный запрос — отменяет подписку.
   
    Платежи:
       - Оплата курсов и отдельных уроков через Stripe.
       - После запроса на оплату вы получаете ссылку, по которой нужно перейти для завершения платежа. 
       - Все платежи сохраняются в личном кабинете и доступны для просмотра.
       - Поддерживается фильтрация платежей по курсу, уроку или способу оплаты.
   
    Роли и права доступа:
       - Обычный пользователь: 
            Создавать свои курсы и уроки, оплачивать чужие, управлять подписками. Видит только свои материалы в списке.
       - Модератор: 
            Просматривать и редактировать все курсы и уроки. Не может удалять и не может создавать новые.
       - Модератор:
             Полный доступ ко всем функциям: создание, редактирование, удаление любого контента.

4. **Авторизация:** 
   ```bash
   Для доступа к защищённым эндпоинтам добавьте заголовок:
   Authorization: Bearer <ваш_jwt_токен> 

5. **Технические особенности:** 
   ```bash
    Безопасность: JWT-аутентификация, валидация всех входных данных.
    Платежи: интеграция с Stripe (тестовые платежи в sandbox-режиме).
    Медиа: загрузка аватаров, превью курсов и уроков.
    Производительность: пагинация на всех списках, фильтрация платежей.
    Валидация: ссылки на видео проверяются на принадлежность YouTube.
